# 침대 카테고리 판매 수량 집계 가이드

니스툴그로우 셀메이트 CSV 데이터를 기준으로 침대류 제품의 판매 **대수**와 카테고리별 **비중 변화**를 분석하기 위한 가이드입니다.

이 문서는 단순 상품명 합산이 아니라, 실제 판매 대수에 가깝게 계산하기 위한 규칙을 정의합니다. 특히 침대류는 한 대가 여러 박스/부품 SKU로 나뉘어 출고되므로, 무작정 합산하면 실제 판매량보다 2~4배 크게 집계될 수 있습니다.

---

## 1. 분석 목적

분석의 핵심은 제품별 단순 판매 랭킹이 아니라, 아래 비중 변화를 추적하는 것입니다.

### 1-1. 아기침대 카테고리

```text
아기침대 = 아이비 + 네티류
```

중요한 지표:

```text
아이비 비중 = 아이비 / 아기침대 전체
네티류 비중 = 네티류 / 아기침대 전체
```

### 1-2. 주니어 침대 카테고리

```text
주니어 침대 = 헤이 + 캠퍼 + 스틴스류
```

중요한 지표:

```text
헤이 비중 = 헤이 / 주니어 침대 전체
캠퍼 비중 = 캠퍼 / 주니어 침대 전체
스틴스류 비중 = 스틴스류 / 주니어 침대 전체
```

### 1-3. 주니어 침대 구조별 비중

브랜드와 별도로, 주니어 침대는 구조 변화가 중요합니다.

```text
주니어 구조 = 싱글 + 로우벙커 + 미드 + 하이 + 이층
```

중요한 지표:

```text
싱글 비중 = 싱글 / 주니어 구조 전체
로우벙커 비중 = 로우벙커 / 주니어 구조 전체
미드 비중 = 미드 / 주니어 구조 전체
하이 비중 = 하이 / 주니어 구조 전체
이층 비중 = 이층 / 주니어 구조 전체
```

## 2. 공통 원칙

### 2-1. 배송유형은 합산

아래 배송유형 prefix는 같은 제품으로 봅니다.

```text
자체
현대물류용
에바다용
```

예:

```text
자체 하이가드 유아침대 아이비 (Ivy)
현대물류용 하이가드 유아침대 아이비 (Ivy)
에바다용 하이가드 유아침대 아이비 (Ivy)
```

위 세 상품은 배송유형만 다르므로, 제품 대수 계산에서는 합산합니다.

단, 계산은 먼저 배송유형별로 정확히 한 뒤 합산하는 것이 안전합니다. 배송유형별 SKU 구성이나 누락 시점이 다를 수 있기 때문입니다.

### 2-2. 옵션은 무조건 합산하지 않음

옵션이 색상 옵션인지, 박스 번호인지, 확장키트인지, 사다리인지 구분해야 합니다.

잘못된 방식:

```text
1번 + 2번 + 3번 + 사다리 + 확장키트 전체 합산
```

올바른 방식:

```text
필수 구성품 중 가장 적게 출고된 수량 = 실제 완성 대수
```

대부분의 세트형 침대는 `min()` 기준을 사용합니다.

### 2-3. 기본 제외 항목

아래 항목은 기본 판매 대수에서 제외합니다.

```text
B급
AS용
CS팀
검수건
모듈변경
범퍼가드
펠트도어
하단 서랍
와이드 하단 서랍
안전가드
침구
매트리스
커버
패드
통판 1ea
증정품
```

단, `모듈`이라는 단어가 들어간다고 무조건 제외하면 안 됩니다. 일부 정상 판매 옵션에 `모듈전체공용` 같은 표현이 들어갈 수 있습니다.

명확히 제외해야 하는 모듈변경 항목:

```text
아이비SS -＞ 캠퍼 모듈변경
```

이 항목은 캠퍼 판매량에 섞지 않고, 별도 `아이비→캠퍼 전환키트`로 집계합니다.

## 3. 아기침대 카테고리

### 3-1. 포함 제품

아기침대 카테고리는 아래 제품군으로 구성합니다.

```text
아이비
네티
네티 플러스
네티 맥스
```

### 3-2. 제외 제품

아기침대 본체 판매량에서는 아래를 제외합니다.

```text
아이비 범퍼가드
아이비 펠트도어
아이비 와이드 하단 서랍
네티/네티플러스 와이드 하단 서랍
네티맥스 범퍼가드
유아용 안티 진드기커버
AS용
B급
CS팀
```

이들은 본체가 아니라 액세서리, 부품, 예외 출고입니다.

## 4. 아이비 집계

아이비 계산 규칙은 이 문서 안의 기준을 적용합니다. 과거 아이비 단독 가이드의 핵심 내용은 아래 규칙으로 통합되었습니다.

### 4-1. 포함 기준

상품명에 아래 중 하나가 포함됩니다.

```text
아이비
Ivy
```

단, 본체만 포함합니다.

### 4-2. 제외 기준

```text
범퍼가드
펠트도어
AS용
모듈변경
B급
통판 1세트
통판 1ea
와이드 하단 서랍
```

### 4-3. 판매 대수 계산

SS:

```text
아이비 SS 대수 = min(SS 박스1 전체, SS 박스2 전체)
```

ES:

```text
아이비 ES 대수 = min(ES 1, ES 2)
```

전체:

```text
아이비 전체 = 아이비 SS + 아이비 ES
```

### 4-4. 색상별 분석

아이비 SS의 색상별 분석은 박스1 기준으로 봅니다.

```text
소프트화이트 = SS 1 소프트화이트
라떼베이지 = SS 1 라떼베이지
구형 화이트 = SS 1 또는 1
```

박스2는 `소프트화이트/라떼베이지 공용`인 경우가 많으므로 색상별로 나누지 않습니다.

## 5. 네티류 집계

### 5-1. 네티

상품명:

```text
네티
```

옵션 예:

```text
화이트&아몬드
화이트&오크
```

계산:

```text
네티 대수 = 옵션 수량 그대로 합산
```

네티는 데이터상 단품형으로 보이므로 `min()` 처리하지 않습니다.

### 5-2. 네티 플러스

상품명:

```text
네티 플러스 (니스툴그로우 2in1 아기침대)
```

옵션 예:

```text
화이트&아몬드
화이트&오크
소프트화이트&아몬드
소프트화이트&오크
소프트화이트&아몬드 (드랍사이드 ver)
소프트화이트&오크 (드랍사이드 ver)
```

계산:

```text
네티 플러스 대수 = 완제품 옵션 수량 그대로 합산
```

주의:

```text
드랍사이드 변형키트
```

는 완제품이 아니라 변형키트이므로 본체 판매량에서 제외하고 별도 집계합니다.

### 5-3. 네티 맥스

상품명:

```text
네티 맥스
```

옵션 예:

```text
소프트화이트&아몬드 1번
소프트화이트&아몬드 2번
소프트화이트&오크 1번
소프트화이트&오크 2번
```

계산:

```text
네티 맥스 대수 = 색상별 min(1번, 2번)
```

예:

```text
소프트화이트&아몬드 = min(아몬드 1번, 아몬드 2번)
소프트화이트&오크 = min(오크 1번, 오크 2번)
```

## 6. 주니어 침대 카테고리

### 6-1. 포함 제품

```text
헤이
캠퍼
스틴스류
```

### 6-2. 주니어 구조 분류

주니어 침대는 아래 구조로 재분류합니다.

```text
싱글
로우벙커
미드
하이
이층
```

브랜드별 비중과 구조별 비중을 모두 봅니다.

주의:

```text
미드 침대 = 싱글 본체 1대 + 미드 확장 구성
하이 침대 = 싱글 본체 1대 + 하이 확장 구성
이층 침대 = 싱글 본체 2대 + 이층 확장 구성
```

따라서 구조별 비중에서 `싱글`은 단순 싱글 본체 출고량이 아니라 **단독 싱글 침대**만 의미합니다.

```text
단독 싱글 = 싱글 본체 출고량 - 미드 - 하이 - (이층 × 2)
```

값이 음수가 되면 0으로 처리합니다. 로우벙커는 싱글 본체를 차감하지 않는 별도 단독 모델로 분리합니다.

## 7. 캠퍼 집계

### 7-1. 캠퍼 미드

상품명:

```text
캠퍼(Camper) 미드
```

주요 옵션 구조:

```text
캠퍼단층SS ... 1번
캠퍼단층SS ... 2번
캠퍼단층SS ... 3번
캠퍼SS 미드확장키트 ...
캠퍼SS 미드 수납형 사다리 ...
```

계산 원칙:

```text
캠퍼 미드 대수 = min(단층 본체 구성, 미드확장키트, 미드 사다리)
```

구형 3박스 구조:

```text
단층 본체 구성 = min(1번, 2번, 3번)
```

통판버전:

```text
단층 본체 구성 = min(1번 전체, 2번 전체)
```

색상별로 보려면 1번 수량을 기준으로 삼되, 실제 대수는 필수 구성품 병목값을 확인합니다.

### 7-2. 캠퍼 하이

상품명:

```text
캠퍼(Camper) 하이확장키트
```

주요 옵션 구조:

```text
소프트화이트 1번
소프트화이트 2번
라떼베이지 1번
라떼베이지 2번
캠퍼SS 하이 수납형 사다리 ...
캠퍼SS 하이 일자사다리
```

계산:

```text
캠퍼 하이확장 판매량 = min(하이 1번 전체, 하이 2번 전체, 하이 사다리)
```

주의:

캠퍼 하이는 완성 침대 전체라기보다 **하이확장키트 판매량**으로 보는 것이 안전합니다. 캠퍼 본체는 미드/단층 구성에 이미 포함될 수 있습니다.

### 7-3. 캠퍼 2층

상품명:

```text
헤이(HEJ), 캠퍼(Camper) 공용 2층확장키트 SS
```

캠퍼 관련 옵션:

```text
1번 - 소프트화이트/베이지 공용
2번 - 소프트화이트/베이지 공용
캠퍼용 짧은키트 (2개매칭)
```

계산:

```text
캠퍼 2층확장 판매량 = min(1번, 2번, 캠퍼용 짧은키트 / 2)
```

`캠퍼용 짧은키트`는 1대당 2개 매칭이므로 2로 나눕니다.

### 7-4. 아이비→캠퍼 전환키트 분리

상품명:

```text
아이비SS -＞ 캠퍼 모듈변경
```

옵션 예:

```text
캠퍼 전면 긴 간살가드
캠퍼 헤드판 -소프트화이트
```

계산:

```text
아이비→캠퍼 전환키트 = min(캠퍼 전면 긴 간살가드, 캠퍼 헤드판)
```

주의:

이 항목은 캠퍼 미드/하이/2층 판매량에 절대 섞지 않습니다. 실제 캠퍼 신규 판매가 아니라 아이비를 캠퍼 구조로 바꾸는 전환 키트입니다.

## 8. 헤이 집계

### 8-1. 헤이 싱글

상품명:

```text
헤이(HEJ) 단층침대 SS
헤이(HEJ) 단층침대 ES
```

옵션 구조:

```text
1번
2번
3번(일부 통판버전)
```

계산:

```text
헤이 싱글 대수 = min(1번 전체, 2번 전체)
```

`3번`은 통판/공용 성격이 있을 수 있으므로 기본 본체 대수 계산에서는 1번/2번을 우선 기준으로 합니다.

### 8-2. 헤이 로우벙커

상품명:

```text
헤이(HEJ) 로우벙커 침대 SS (라이즈)
헤이(HEJ) 로우벙커 침대 ES (라이즈)
```

옵션 구조:

```text
소프트화이트 1번
소프트화이트 2번
```

계산:

```text
헤이 로우벙커 대수 = min(1번, 2번)
```

### 8-3. 헤이 미드

상품명:

```text
헤이(HEJ) 미드확장키트 SS
헤이(HEJ) 미드확장키트 ES
```

옵션 구조:

```text
화이트 3번
화이트 4번
소프트화이트
베이지
```

구형 계산:

```text
헤이 미드 대수 = min(3번, 4번)
```

신형 계산:

```text
헤이 미드 대수 = 미드확장키트 색상 옵션 수량
```

주의:

미드는 완성 침대 전체가 아니라 **미드확장키트 판매량**으로 봅니다.

### 8-4. 헤이 하이

상품명:

```text
헤이(HEJ) 하이확장키트 SS
헤이(HEJ) 하이확장키트 ES
```

옵션 구조:

```text
1번
2번
```

계산:

```text
헤이 하이 대수 = min(1번, 2번)
```

하이는 완성 침대 전체가 아니라 **하이확장키트 판매량**으로 봅니다.

### 8-5. 헤이 2층

상품명:

```text
헤이(HEJ) 2층확장키트 ES
헤이(HEJ), 캠퍼(Camper) 공용 2층확장키트 SS
```

헤이 관련 옵션:

```text
1번 - 소프트화이트/베이지 공용
2번 - 소프트화이트/베이지 공용
헤이용 짧은키트 (1개매칭)
짧은키트 (1개매칭)
```

계산:

```text
헤이 2층 대수 = min(1번, 2번, 헤이용 짧은키트)
```

ES 전용 2층확장키트도 동일하게:

```text
헤이 ES 2층 대수 = min(1번, 2번, 짧은키트)
```

## 9. 스틴스류 집계

스틴스류는 주니어 침대 카테고리에 포함합니다.

스틴스는 시기별로 SKU 이름이 다르게 들어가 있습니다.

```text
구형 구조 SKU: 싱글침대, 미드확장키트, 하이확장키트, 벙커확장키트
신형 구조 SKU: DP용 스틴스 / 싱글침대, 미드침대, 하이침대, 2층침대 옵션
```

주의:

```text
스틴스 완장 / 단일상품
스틴스 쿠키
스틴스 안전가드
```

위 항목들은 침대 구조 계산에 사용하지 않습니다. 특히 `스틴스 완장 / 단일상품`은 과거 판매량이 크지만, 2020년 데이터를 확인하면 별도로 `싱글침대`, `미드확장키트`, `하이확장키트`, `벙커확장키트`가 존재하므로 완장을 스틴스 총량이나 싱글로 잡으면 중복 또는 오분류가 됩니다.

### 9-1. 구형 스틴스 포함 기준

상품명:

```text
싱글침대
미드확장키트
하이확장키트
벙커확장키트
현대물류용 싱글침대
현대물류용 미드확장키트
현대물류용 하이확장키트
현대물류용 벙커확장키트
에바다용 싱글침대
에바다용 미드확장키트
에바다용 하이확장키트
에바다용 벙커확장키트
```

구형 스틴스 색상 옵션 예:

```text
화이트워시
투톤(스톤)
연그레이
```

구형 구조 분류:

```text
스틴스 싱글 본체 = 상품명 '싱글침대'
스틴스 미드 = 상품명 '미드확장키트'
스틴스 하이 = 상품명 '하이확장키트'
스틴스 2층 = 상품명 '벙커확장키트'
```

`벙커확장키트`는 스틴스 2층 구조로 봅니다.

### 9-2. 신형 DP용 스틴스 포함 기준

상품명:

```text
DP용 스틴스
```

옵션 예:

```text
싱글침대 투톤
싱글침대 화이트워시
싱글침대 연그레이
싱글가드침대 투톤
싱글가드침대 화이트워시
싱글가드침대 연그레이
미드침대 투톤
미드침대 화이트워시
미드침대 연그레이
하이침대 투톤
하이침대 화이트워시
하이침대 연그레이
2층침대 투톤
2층침대 화이트워시
2층침대 연그레이
```

신형 구조 분류:

```text
스틴스 싱글 본체 = 옵션명에 '싱글침대' 또는 '싱글가드침대' 포함
스틴스 미드 = 옵션명에 '미드침대' 포함
스틴스 하이 = 옵션명에 '하이침대' 포함
스틴스 2층 = 옵션명에 '2층침대' 또는 '이층침대' 포함
```

`싱글가드침대`는 별도 구조가 아니라 싱글 침대에 전면 가드가 붙은 형태로 봅니다. 따라서 `싱글침대`와 통합하여 싱글 본체 수량에 포함합니다.

### 9-3. 제외 기준

```text

DP용 스틴스 완장

침구
커버
패드
```

### 9-4. 단독 싱글 계산

스틴스의 미드, 하이, 2층 판매에는 싱글 본체가 함께 포함됩니다.

```text
미드 1대 = 싱글 본체 1대 + 미드 구성
하이 1대 = 싱글 본체 1대 + 하이 구성
2층 1대 = 싱글 본체 2대 + 2층 구성
```

따라서 구조별 비중에서 `싱글`은 전체 싱글 본체 출고량이 아니라, 다른 구조에 포함되지 않은 **단독 싱글 침대**만 의미합니다.

계산식:

```text
스틴스 단독 싱글 = 스틴스 싱글 본체 - 스틴스 미드 - 스틴스 하이 - (스틴스 2층 × 2)
```

음수가 나오면 0으로 처리합니다.

```text
스틴스 단독 싱글 = max(0, 스틴스 싱글 본체 - 스틴스 미드 - 스틴스 하이 - (스틴스 2층 × 2))
```

중요 메모:

```text
과거 스틴스 판매 시기에는 이미 싱글침대를 보유한 고객이 나중에 미드확장키트, 하이확장키트, 벙커확장키트만 별도로 구매해 침대를 변형한 경우가 있었다.
```

현재 CSV에는 아래 구분이 별도 상품명/옵션명으로 명확히 남아 있지 않습니다.

```text
신규 완성 침대용 확장키트
기존 싱글 고객 업그레이드용 확장키트
확장키트만 구매
```

따라서 현재 공식은 실제 단독 싱글 판매량을 정확히 계산한다기보다, 월별 데이터 안에서 중복을 피하기 위한 **보수적 하한값**입니다.

이 방식의 한계:

```text
기존 고객이 확장키트만 구매한 경우에도 미드/하이/2층 판매량에는 잡힌다.
하지만 같은 달 싱글 본체 판매에서 차감할 싱글이 없을 수 있다.
이 경우 단독 싱글은 과소계산되고, 미드/하이/2층은 신규 완성침대 판매가 아니라 기존 고객 확장 수요까지 포함한 값이 된다.
```

데이터에서 확인된 간접 신호:

```text
2017.02: 싱글 본체 299 / 확장 구조 필요 싱글 환산 384
2018.01: 싱글 본체 455 / 확장 구조 필요 싱글 환산 572
2021.12: 싱글 본체 430 / 확장 구조 필요 싱글 환산 505
2022.12: 싱글 본체 114 / 확장 구조 필요 싱글 환산 157
```

향후 재계산 필요:

```text
스틴스는 "신규 완성 침대 판매량"과 "기존 고객 확장/업그레이드 수요"를 분리하는 별도 기준이 필요하다.
가능하면 주문 단위 원장, 고객 재구매 여부, 세트 매칭 주문번호, 또는 같은 주문 내 싱글 본체 동시 구매 여부를 확인해야 한다.
현재 월별 SKU 집계 CSV만으로는 이 둘을 정확히 분리할 수 없다.
```

그래프 해석:

```text
스틴스 미드/하이/2층은 신규 완성침대 판매라기보다 해당 구조로 전환 또는 확장된 수요까지 포함한 값으로 해석한다.
스틴스 단독 싱글은 실제 단독 싱글 판매량의 하한값으로 본다.
```

### 9-5. 스틴스 전체 판매 대수

스틴스 전체 판매 대수는 최종 침대 형태 기준으로 합산합니다.

```text
스틴스 전체 = 스틴스 단독 싱글 + 스틴스 미드 + 스틴스 하이 + 스틴스 2층
```

예: 2020년 전체 배송형태 합산 기준

```text
스틴스 싱글 본체 = 7,481
스틴스 미드 = 2,319
스틴스 하이 = 929
스틴스 2층 = 1,916
스틴스 단독 싱글 = 7,481 - 2,319 - 929 - (1,916 × 2) = 401
스틴스 전체 = 401 + 2,319 + 929 + 1,916 = 5,565
```

주의:

```text
스틴스 싱글 본체 전체를 그대로 더하면 미드/하이/2층에 들어간 싱글 본체가 중복 계산됩니다.
스틴스 전체 판매량에는 싱글 본체 전체가 아니라 차감 후 단독 싱글만 넣습니다.
스틴스 완장 / 단일상품은 구조 계산에서 제외합니다.
```

## 10. 데스크 집계

데스크는 시기별 SKU 구조가 다릅니다. 같은 "책상 전체"라도 어느 시기에는 다리 수량이 가장 정확하고, 어느 시기에는 상판 수량이 가장 정확합니다.

### 10-1. 전체 데스크 수량

#### 2018년 이전: 구형 그로우 데스크

2014~2017년 구간은 `상판SN/CN` 판매값이 거의 없거나 부분적으로만 잡힙니다. 반면 `다리W/G`와 `리어탑SN/CN`은 거의 1:1로 움직입니다.

따라서 2018년 이전 구형 그로우 데스크 전체 수량은 `다리W + 다리G`를 기준으로 봅니다.

상품명:

```text
그로우 데스크
현대물류용 그로우 데스크
에바다용 그로우 데스크
```

옵션:

```text
다리W (구형)
다리G (구형)
다리W (패킹변경)
다리G (패킹변경)
```

계산:

```text
2018년 이전 그로우 데스크 전체 = 다리W + 다리G
```

검증 예:

```text
2015년: 다리W+G 6,660 / 리어탑SN+CN 6,675 / 상판SN+CN 32
2016년: 다리W+G 6,712 / 리어탑SN+CN 6,737 / 상판SN+CN 30
2017년: 다리W+G 5,963 / 리어탑SN+CN 5,973 / 상판SN+CN 509
```

이 구간에서 `상판SN+CN`을 전체 수량으로 쓰면 과소계산됩니다.

#### 2018년 이후: 그로우 데스크 상판 기준

2018년부터는 `상판SN/CN`, `다리W/G`, `리어탑SN/CN`이 함께 잡히기 시작합니다. 이때부터는 상판 모양 구분인 스탠다드와 커브를 합산해 전체 수량으로 봅니다.

옵션:

```text
상판SN
상판SN (패킹변경)
상판CN
상판CN (패킹변경)
```

계산:

```text
2018년 이후 그로우 데스크 전체 = 상판SN + 상판CN
```

검증 예:

```text
2018년: 상판SN+CN 6,763 / 리어탑SN+CN 6,907 / 다리W+G 6,952
2020년: 상판SN+CN 9,455 / 리어탑SN+CN 9,374
```

#### 현행 뉴트럴 데스크

현행 뉴트럴 데스크는 프레임 수량으로 전체 수량을 봅니다.

상품명:

```text
뉴트럴 플라이우드 데스크
현대물류용 뉴트럴 플라이우드 데스크
에바다용 뉴트럴 플라이우드 데스크
```

옵션:

```text
프레임-오프화이트
```

계산:

```text
현행 뉴트럴 데스크 전체 = 프레임-오프화이트 수량
```

단, 2021년 말처럼 뉴트럴 데스크 전환기에는 `프레임-오프화이트` 값이 거의 없고 `상판-*`, `리어탑-*` 수량만 정상적으로 잡힌 달이 있습니다. 이 구간에서 프레임만 사용하면 전체 수량이 과소계산됩니다.

전환기 보정:

```text
뉴트럴 데스크 보정 수량 = max(프레임-오프화이트, min(상판 전체, 리어탑 전체))
```

예:

```text
2021년 12월:
그로우 상판SN/CN = 55
뉴트럴 프레임 = 3
뉴트럴 상판 = 539
뉴트럴 리어탑 = 537
기존 프레임 기준 = 55 + 3 = 58
보정 기준 = 55 + max(3, min(539, 537)) = 592
```

따라서 현행 뉴트럴 구간의 전체 수량은 원칙적으로 프레임을 우선하되, 프레임 기록이 누락된 전환기는 상판/리어탑 병목 수량으로 보정합니다.

최종 데스크 전체:

```text
데스크 전체 = 2018년 이전 그로우 데스크 전체 + 2018년 이후 그로우 데스크 전체 + 현행 뉴트럴 데스크 전체/전환기 보정 수량
```

배송유형은 합산합니다.

### 10-2. 상판 옵션 구조

상판은 옵션 구성 분석용입니다.

2018년 이전 그로우 데스크는 상판 데이터가 충분하지 않으므로, 상판 구성 비중을 정확히 나누지 않습니다. 이 구간의 플라이우드 상판 전체는 책상 전체 수량과 같은 값으로 보되, 스탠다드/커브 비중 분석에는 사용하지 않습니다.

2018년 이후 그로우 데스크의 플라이우드 상판:

```text
상판SN
상판CN
```

`SN`과 `CN`은 상판 모양 구분입니다. 색상 전체 수량에서는 둘을 합산합니다.

현행 뉴트럴 데스크의 플라이우드 색상 옵션:

```text
상판-유로화이트
상판-오트밀
상판-로즈레이스
상판-프레스콧그린
상판-내추럴
상판-월넛
```

오크원목:

```text
상판-오크솔리드원목
```

### 10-3. 리어탑 옵션 구조

리어탑도 옵션 구성 분석용입니다.

과거 그로우 데스크의 플라이우드 리어탑:

```text
리어탑SN
리어탑CN
```

현행 뉴트럴 데스크의 플라이우드 색상 옵션:

```text
리어탑-유로화이트
리어탑-오트밀
리어탑-로즈레이스
리어탑-프레스콧그린
리어탑-내추럴
리어탑-월넛
```

오크원목:

```text
리어탑-오크솔리드원목
```

### 10-4. 오크원목 데스크 추정

오크원목 데스크는 현행 뉴트럴 데스크 안의 옵션 구조입니다.

계산:

```text
오크원목 데스크 추정 수량 = min(상판-오크솔리드원목, 리어탑-오크솔리드원목)
```

주의:


## 12. 주의점 요약

```text
아이비는 이 문서의 박스1/박스2 min 기준과 색상별 박스1 기준을 우선 적용.

캠퍼 하이, 헤이 미드/하이/2층은 완성 침대가 아니라 확장키트 판매량으로 해석.

캠퍼용 짧은키트는 2개매칭이므로 2로 나눔.
스틴스 단독 싱글은 싱글 본체에서 미드, 하이, 2층×2를 뺀 값으로 계산.
아이비SS -> 캠퍼 모듈변경은 캠퍼 판매량에서 제외하고 별도 집계.

액세서리, B급, AS용, CS팀, 부품은 본체 판매량에서 제외.
```

```text
아기침대는 아이비 vs 네티류 비중을 본다.
```
