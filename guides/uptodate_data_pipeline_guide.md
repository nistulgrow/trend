# Uptodate 데이터 업데이트 절차 가이드

니스툴그로우 셀메이트 CSV 원본 데이터를 기준으로 `uptodate_YYYYMMDD.csv` 및 주간/월간/분기/연간 집계 파일을 만드는 표준 절차입니다.

이 문서는 추후 같은 작업을 반복할 때 실수를 줄이기 위한 운영 가이드입니다. 핵심 원칙은 **기존 uptodate 파일을 기준 원장으로 유지**, **신규 raw 파일의 추가 날짜만 반영**, **상품명+옵션내용 기준 병합**, **검증 후 input 교체**입니다.

---

## 1. 폴더 구조

작업 기준 폴더:

```text
thewalldata/
  rawdata/
    uptodate_YYYYMMDD.csv
    신규 raw CSV 파일들
    old/
      처리 완료된 과거 원본 CSV 보관
  input/
    분석용 최신 uptodate 파일들
  guides/
    작업 절차 및 분석 가이드
```

최종 처리 후 기대 구조:

```text
rawdata/
  uptodate_YYYYMMDD.csv
  old/
    과거 uptodate 파일
    처리 완료된 신규 raw 파일들

input/
  uptodate_YYYYMMDD.csv
  uptodate_YYYYMMDD_week.csv
  uptodate_YYYYMMDD_month.csv
  uptodate_YYYYMMDD_quater.csv
  uptodate_YYYYMMDD_year.csv
```

`rawdata` 루트에는 최신 `uptodate_YYYYMMDD.csv`와 아직 처리하지 않은 신규 raw 파일만 있어야 합니다.

---

## 2. 전체 작업 흐름

1. `rawdata` 폴더에서 기존 `uptodate_YYYYMMDD.csv`를 찾는다.
2. 기존 uptodate 파일의 마지막 날짜를 확인한다.
3. `rawdata` 폴더에 새로 들어온 raw CSV 파일을 찾는다.
4. 신규 raw 파일의 실제 날짜 컬럼 범위를 확인한다.
5. 기존 uptodate의 마지막 날짜보다 늦은 날짜 컬럼만 추출한다.
6. 기존 기간과 겹치는 날짜는 무시하고 기존 uptodate 값을 그대로 둔다.
7. 신규 날짜 값은 `(상품명, 옵션내용)` 키 기준으로 기존 uptodate에 붙인다.
8. 신규 raw에만 있는 상품/옵션 키는 새 행으로 추가한다.
9. `현재고`, `미발송수`는 가장 최근 raw 파일의 값으로 갱신한다.
10. `주문수`, `주문금액`은 빈칸으로 둔다.
11. 새 일별 파일 `rawdata/uptodate_YYYYMMDD.csv`를 만든다.
12. 기존 기간 보존과 신규 기간 반영을 검증한다.
13. 검증 완료 후 `input` 폴더의 과거 uptodate 파일을 삭제한다.
14. 새 일별 파일을 `input` 폴더로 복사한다.
15. 새 일별 파일에서 주간/월간/분기/연간 집계 파일을 다시 만든다.
16. 처리한 신규 raw 파일과 이전 uptodate 파일을 `rawdata/old`로 이동한다.

## 3. 대상 파일 확인

### 3-1. 기존 기준 파일

`rawdata` 루트에서 기존 기준 파일을 찾습니다.

```text
rawdata/uptodate_YYYYMMDD.csv
```

예:

```text
rawdata/uptodate_20260610.csv
```

이 파일이 이번 업데이트의 기준 원장입니다.

### 3-2. 신규 raw 파일

`rawdata` 루트에 새로 들어온 CSV 파일을 찾습니다.

대상:

```text
rawdata/*.csv
```

제외:

- `rawdata/uptodate_*.csv`
- `rawdata/old/*`
- 숨김 파일
- 임시 파일

예:

```text
rawdata/stk_optOrdProgress_20260702_051715.csv
```

### 3-3. 날짜 기준

파일명은 참고만 합니다. 실제 기준은 반드시 CSV 내부의 날짜 컬럼입니다.

날짜 컬럼 조건:

- 컬럼명이 숫자 8자리
- 형식: `YYYYMMDD`
- 예: `20260702`

최종 파일명 날짜는 기존 uptodate와 신규 raw 파일을 합친 뒤의 마지막 날짜입니다.

예:

```text
기존 uptodate 마지막 날짜: 20260610
신규 raw 마지막 날짜: 20260702
최종 파일명: uptodate_20260702.csv
```

## 4. 인코딩 기준

### 4-1. 원본 읽기

셀메이트 raw CSV는 `EUC-KR` 또는 `CP949`로 읽습니다.

이유:

- 원본 파일의 한글 컬럼명과 상품명이 EUC-KR 계열입니다.
- UTF-8로 바로 읽으면 한글이 깨질 수 있습니다.

기존 `uptodate_YYYYMMDD.csv`는 결과 파일이므로 `UTF-8 BOM`으로 읽는 것을 기본으로 합니다.

### 4-2. 결과 저장

결과 CSV는 `UTF-8 BOM`으로 저장합니다.

이유:

- Excel에서 바로 열었을 때 한글이 깨지지 않습니다.
- 이후 Python, Google Sheets, Excel 모두에서 다루기 쉽습니다.

## 5. 병합 기준

### 5-1. 절대 원칙

**행 순서는 절대 믿지 않습니다.**

원본 파일마다 상품 행 순서가 다를 수 있습니다. 행 번호 기준으로 붙이면 다른 상품의 주문 수량이 섞입니다.

반드시 아래 두 컬럼을 키로 사용합니다.

```text
상품명
옵션내용
```

즉, 병합 기준은:

```text
(상품명, 옵션내용)
```

입니다.

### 5-2. 날짜 데이터 병합

모든 날짜별 주문 수량은 같은 `(상품명, 옵션내용)` 키에만 붙입니다.

예:

```text
상품명: 자체 하이가드 유아침대 아이비 (Ivy)
옵션내용: SS 1 (통판트레이버전-통판, 사다리 리뉴얼) 소프트화이트
```

위 키에 해당하는 행에는 같은 상품/옵션의 날짜별 수량만 들어가야 합니다.

### 5-3. 신규 상품/옵션 키

신규 raw 파일에 기존 uptodate에 없던 `(상품명, 옵션내용)` 키가 있으면 새 행으로 추가합니다.

새 행의 날짜 값 처리:

- 기존 기간 날짜: 빈칸 또는 0이 아니라, 기존에 데이터가 없었음을 나타내는 빈 값으로 둡니다.
- 신규 raw에 포함된 추가 날짜: raw 파일의 값을 넣습니다.
- `현재고`, `미발송수`: 신규 raw 파일의 값을 넣습니다.
- `주문수`, `주문금액`: 빈칸으로 둡니다.

## 6. 날짜 반영 규칙

### 6-1. 기준 날짜

기존 uptodate 파일의 마지막 날짜를 먼저 확인합니다.

예:

```text
기존 파일: rawdata/uptodate_20260610.csv
기존 마지막 날짜: 20260610
```

### 6-2. 겹치는 기간 처리

신규 raw 파일에 기존 uptodate와 겹치는 날짜가 있어도, 그 구간은 반영하지 않습니다.

예:

```text
기존 uptodate: 20140601 ~ 20260610
신규 raw:      20260602 ~ 20260702
```

이 경우:

```text
20260602 ~ 20260610: 무시, 기존 uptodate 값 보존
20260611 ~ 20260702: 신규 raw 값 추가
```

중복 날짜는 새 raw 파일 값으로 덮어쓰지 않습니다.

### 6-3. 추가 날짜만 append

기존 마지막 날짜보다 늦은 날짜만 새 컬럼으로 추가합니다.

예:

```text
추가 대상 날짜:
20260611
20260612
...
20260702
```

최종 날짜 컬럼은 오름차순으로 정렬합니다.

### 6-4. 시작일 필터

의미 있는 데이터 시작일은 `20140601`입니다.

일별 통합 파일에는 `20140601` 이전 날짜 컬럼을 넣지 않습니다.

삭제 또는 제외 대상:

```text
20140101 ~ 20140531
```

## 7. 일별 통합 파일 생성

### 7-1. 파일명

```text
uptodate_YYYYMMDD.csv
```

예:

```text
uptodate_20260702.csv
```

`YYYYMMDD`는 최종 일별 데이터의 마지막 날짜입니다.

### 7-2. 저장 위치

1차 결과물은 먼저 `rawdata` 폴더에 저장합니다.

```text
rawdata/uptodate_YYYYMMDD.csv
```

검증이 끝난 뒤 `input` 폴더로 복사합니다.

```text
input/uptodate_YYYYMMDD.csv
```

### 7-3. 컬럼 구조

앞 6개 고정 컬럼:

```text
상품명
옵션내용
현재고
미발송수
주문수
주문금액
```

그 뒤에는 날짜 컬럼이 오름차순으로 이어집니다.

```text
20140601
20140602
...
20260702
```

### 7-4. 현재고, 미발송수, 주문수, 주문금액 처리

`현재고`, `미발송수`는 가장 최근 raw 파일의 값을 사용합니다.

가장 최근 raw 파일은 파일명이 아니라 **실제 날짜 컬럼이 가장 늦은 파일**입니다.

예:

```text
rawdata/stk_optOrdProgress_20260702_051715.csv
마지막 날짜: 20260702
```

이 파일에서 같은 `(상품명, 옵션내용)` 키의:

```text
현재고
미발송수
```

값을 가져옵니다.

신규 raw에 없는 기존 상품/옵션은 기존 uptodate의 `현재고`, `미발송수`를 유지합니다.

`주문수`, `주문금액`은 비워 둡니다.

이유:

- 통합 파일에는 날짜별 주문 수량이 이미 들어 있습니다.
- `주문수`, `주문금액`이 특정 raw 파일의 부분 합계인지 전체 합계인지 혼동될 수 있습니다.
- 분석 오류를 막기 위해 빈칸으로 둡니다.

## 8. 자동 검증

일별 통합 파일을 만든 뒤 반드시 검증합니다.

### 8-1. 기존 기간 보존 검증

기존 uptodate 파일에 있던 날짜는 새 파일에서도 값이 같아야 합니다.

검증 단위:

```text
상품명 + 옵션내용 + 기존 날짜
```

예:

```text
20140601 ~ 20260610
```

이 구간은 새 raw 파일과 겹치더라도 기존 값을 보존해야 합니다.

검증 결과:

```text
불일치 0건
```

이어야 합니다.

### 8-2. 신규 기간 반영 검증

기존 마지막 날짜보다 늦은 신규 날짜는 raw 파일 값과 새 uptodate 파일 값이 같아야 합니다.

검증 단위:

```text
상품명 + 옵션내용 + 신규 날짜
```

예:

```text
20260611 ~ 20260702
```

검증 결과:

```text
누락 키 0건
불일치 0건
```

이어야 합니다.

### 8-3. 검증에서 확인할 것

- 기존 uptodate의 날짜 값이 바뀌지 않았는지
- 신규 raw의 추가 날짜가 빠지지 않았는지
- 신규 raw에 있는 모든 상품/옵션 키가 새 파일에 존재하는지
- 중복 날짜가 새 raw 값으로 덮어써지지 않았는지
- 날짜 컬럼이 오름차순인지
- `20140601` 이전 날짜는 의도대로 제외되었는지
- `현재고`, `미발송수`가 최신 raw 파일 기준인지
- `주문수`, `주문금액`이 빈칸인지

검증 실패가 1건이라도 있으면 완료 처리하지 않습니다.

## 9. input 폴더 정리

검증이 완료된 뒤 `input` 폴더의 과거 uptodate 파일을 정리합니다.

### 9-1. 삭제 대상

```text
input/uptodate_*.csv
input/uptodate_*_week.csv
input/uptodate_*_month.csv
input/uptodate_*_quater.csv
input/uptodate_*_year.csv
```

### 9-2. 삭제 순서

안전한 순서:

1. 새 일별 파일을 `rawdata`에 생성
2. 검증 완료
3. 과거 `input/uptodate_*` 파일 삭제
4. 최신 일별 파일을 `input`으로 복사
5. 최신 집계 파일 생성

실패했는데 기존 `input` 파일까지 삭제하면 분석용 파일이 사라지므로, 검증 전에는 삭제하지 않습니다.

최종적으로 `input` 폴더에는 최신 날짜 기준 파일 5개가 남습니다.

예:

```text
input/uptodate_20260702.csv
input/uptodate_20260702_week.csv
input/uptodate_20260702_month.csv
input/uptodate_20260702_quater.csv
input/uptodate_20260702_year.csv
```

## 10. rawdata 폴더 정리

### 10-1. 이동 대상

검증과 `input` 파일 생성이 모두 끝난 뒤, 처리에 사용한 파일을 `rawdata/old`로 이동합니다.

이동 대상:

- 이전 `rawdata/uptodate_YYYYMMDD.csv`
- 이번에 반영한 신규 raw CSV 파일

예:

```text
rawdata/uptodate_20260610.csv                 -> rawdata/old/uptodate_20260610.csv
rawdata/stk_optOrdProgress_20260702_051715.csv -> rawdata/old/stk_optOrdProgress_20260702_051715.csv
```

### 10-2. 남겨야 할 파일

아래 파일은 `rawdata` 루트에 남깁니다.

```text
rawdata/uptodate_20260702.csv
```

### 10-3. old 폴더 충돌 주의

`rawdata/old`에 같은 이름의 파일이 이미 있으면 덮어쓰기 전에 확인해야 합니다.

권장 방식:

- 같은 이름이 없으면 그대로 이동
- 같은 이름이 있으면 덮어쓰기하지 말고 별도 백업명 사용

예:

```text
old/stk_optOrdProgress_20260702_051715.csv
old/stk_optOrdProgress_20260702_051715_backup.csv
```

## 11. 집계 파일 재생성

집계 파일은 반드시 `input/uptodate_YYYYMMDD.csv` 일별 파일에서 다시 만듭니다.

신규 raw 파일에서 바로 주/월/분기/연 파일을 만들지 않습니다.

이유:

- 일별 uptodate 파일이 검증된 기준 데이터입니다.
- 집계 파일들은 모두 같은 기준에서 파생되어야 합니다.
- 중복 날짜 무시 규칙도 일별 uptodate 파일에 이미 반영되어 있어야 합니다.

---

## 12. 주간 파일

### 12-1. 기준

주간 기준은 월요일~일요일입니다.

### 12-2. 컬럼명

```text
YYYYMMDD_YYYYMMDD
```

예:

```text
20260629_20260705
```

### 12-3. 마지막 미완성 주 포함

현재 정책은 **마지막 미완성 주도 포함**입니다.

예를 들어 마지막 일별 데이터가 `20260702`이면:

- 해당 주: `20260629_20260705`
- 실제 포함 데이터: `20260629 ~ 20260702`
- `20260703 ~ 20260705`는 아직 데이터 없음

그래도 해당 주 컬럼을 만들고, 있는 날짜만 합산합니다.

### 12-4. 파일명

```text
uptodate_YYYYMMDD_week.csv
```

예:

```text
uptodate_20260702_week.csv
```

---

## 13. 월간 파일

### 13-1. 기준

월별 컬럼은 `YYYYMM` 형식입니다.

예:

```text
202607
```

### 13-2. 마지막 미완성 월 포함

현재 정책은 **마지막 미완성 월도 포함**입니다.

예를 들어 마지막 일별 데이터가 `20260702`이면:

- `202607` 컬럼 생성
- 실제 포함 데이터: `20260701 ~ 20260702`

### 13-3. 파일명

```text
uptodate_YYYYMMDD_month.csv
```

예:

```text
uptodate_20260702_month.csv
```

## 14. 분기 파일

### 14-1. 기준

분기 컬럼은 `YYYYQn` 형식입니다.

예:

```text
2026Q3
```

### 14-2. 시작 분기

분기 집계는 `2014Q3`부터 시작합니다.

이유:

- 일별 데이터 시작점이 `20140601`입니다.
- `2014Q2`는 6월 한 달만 있어 불완전합니다.
- 안정적인 분기 비교는 `2014Q3`부터가 적절합니다.

### 14-3. 마지막 미완성 분기 포함

현재 정책은 **마지막 미완성 분기도 포함**입니다.

예를 들어 마지막 일별 데이터가 `20260702`이면:

- `2026Q3` 컬럼 생성
- 실제 포함 데이터: `20260701 ~ 20260702`

### 14-4. 파일명

주의: 기존 파일명 정책은 `quarter`가 아니라 `quater`입니다.

```text
uptodate_YYYYMMDD_quater.csv
```

예:

```text
uptodate_20260702_quater.csv
```

## 15. 연간 파일

### 15-1. 기준

연간 컬럼은 `YYYY` 형식입니다.

예:

```text
2026
```

### 15-2. 시작 연도

연간 집계는 `2015`년부터 시작합니다.

이유:

- 2014년 데이터는 6월부터 시작하므로 반년치입니다.
- 연간 비교에서 2014년은 불완전하므로 제외합니다.

### 15-3. 마지막 미완성 연도 포함

현재 정책은 **마지막 미완성 연도도 포함**입니다.

예를 들어 마지막 일별 데이터가 `20260702`이면:

- `2026` 컬럼 생성
- 실제 포함 데이터: `20260101 ~ 20260702`

### 15-4. 파일명

```text
uptodate_YYYYMMDD_year.csv
```

예:

```text
uptodate_20260702_year.csv
```

## 17. 반드시 지켜야 할 주의점

### 17-1. 중복 날짜는 덮어쓰지 않기

신규 raw 파일이 기존 uptodate와 날짜가 겹치면, 겹치는 날짜는 무시합니다.

기존 파일:

```text
~ 20260610
```

신규 raw:

```text
20260602 ~ 20260702
```

반영:

```text
20260611 ~ 20260702만 추가
```

기존 `20260602 ~ 20260610` 값은 그대로 둡니다.

### 17-2. 행 순서 기준 병합 금지

가장 중요한 주의점입니다.

원본 파일들은 행 수가 같아 보여도 상품 순서가 다를 수 있습니다.

잘못된 방식:

```text
1번째 행에는 1번째 행 날짜값 붙이기
2번째 행에는 2번째 행 날짜값 붙이기
```

올바른 방식:

```text
상품명 + 옵션내용이 같은 행에만 날짜값 붙이기
```

### 17-3. 파일명만 믿지 않기

파일명이 날짜를 암시하더라도 반드시 CSV 내부의 날짜 컬럼을 확인합니다.

과거에 파일명과 실제 날짜 컬럼이 맞지 않는 사례가 있었습니다.

### 17-4. 검증 전 파일 이동 금지

검증이 끝나기 전에는 신규 raw 파일과 기존 uptodate 파일을 `old`로 이동하지 않습니다.

검증 실패 시 원본 파일이 흩어지면 재작업이 번거로워집니다.

### 17-5. 현재고/미발송수는 합산하지 않기

`현재고`, `미발송수`는 기간 합계가 아닙니다.

가장 최근 raw 파일의 현재 상태값만 사용합니다.

신규 raw에 없는 기존 상품/옵션은 기존 uptodate 값을 유지합니다.

### 17-6. 주문수/주문금액은 비워 두기

통합 파일과 집계 파일에서 `주문수`, `주문금액`은 빈칸으로 둡니다.

이 컬럼에 특정 raw 파일의 합계가 남아 있으면 전체 기간 합계처럼 오해할 수 있습니다.

### 17-7. 마지막 기간은 버리지 않기

현재 정책은 up to date 방식입니다.

따라서 마지막 주/월/분기/연도가 끝나지 않았더라도, 있는 날짜까지 합산해서 포함합니다.

### 17-8. old 폴더 덮어쓰기 주의

`rawdata/old`에 같은 이름의 파일이 있으면 무조건 덮어쓰지 않습니다.

특히 같은 파일명이지만 내용이 수정된 경우가 있을 수 있습니다.

### 17-9. input 과거 파일 삭제 순서 주의

과거 `input/uptodate_*` 파일은 삭제하되, 최신 파일 생성과 검증이 끝난 뒤 정리합니다.

실패했는데 기존 파일까지 삭제하면 분석용 파일이 사라집니다.
